## Prerequisites

- Node.js and pnpm installed
- A SolidJS project (Vite or similar)
- Tailwind CSS (v4) setup — see official docs: https://tailwindcss.com/docs
- Ark UI for SolidJS

## 1) Install Tailwind CSS

Follow Tailwind’s guide for your framework. Typical steps:

```bash
pnpm add -D tailwindcss
npx tailwindcss init
```

Add Tailwind to your global stylesheet:

```css
@import "tailwindcss";
```

Ensure Tailwind scans your Solid source files (adjust paths as needed):

```txt
src/**/*.{ts,tsx}
```

## 2) Install Ark UI (Solid)

```bash
pnpm add @ark-ui/solid
```

Ark UI provides accessible primitives and logic; arkitect-ui builds styled components using Tailwind on top of these primitives.

## 3) Add arkitect-ui components

Choose your workflow: CLI or Manual.

<Tabs>
  <TabItem label="CLI (Recommended)">

Run the initializer to scaffold config and components:

```bash
npx @arkitect-ui/cli init
```

Add specific components when you need them:

```bash
npx @arkitect-ui/cli add button --framework=solid
```

This copies the component source into your project so you have full control (styles, variants, code).

  </TabItem>
  <TabItem label="Manual">

Copy components directly from the documentation (e.g., Button) into `src/components/ui/`. Import and use:

```tsx
import { Button } from "@/components/ui/button";

export function Example() {
  return <Button variant="default">Click me</Button>;
}
```

Manual copy-paste gives you complete ownership over styles and behavior, inspired by shadcn/ui.

  </TabItem>
</Tabs>

## 4) Verify setup

- Import your global stylesheet (Tailwind) once in your app entry
- Render a sample component (Button) to confirm styles and behavior
